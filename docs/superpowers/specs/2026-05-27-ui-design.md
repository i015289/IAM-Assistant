# IAM Assistant Web UI — Design Spec
Date: 2026-05-27

## Overview

A browser-based chat interface for the IAM Assistant. Replaces the Claude Code terminal for day-to-day IAM analysis, making the tool accessible to teammates on the internal network without requiring a local Claude Code setup.

**Goal:** Chat left / results right layout, served by a Python FastAPI backend, authenticated via corporate SSO (OIDC), deployed on an internal network host.

---

## Architecture

```
iam-assistant/
  app/
    main.py          # FastAPI app entry point — mounts routes, serves static UI
    chat.py          # Anthropic streaming client + MCP tool execution loop
    auth.py          # OIDC authorization code flow (authlib)
    mcp_client.py    # Subprocess wrapper around er6_mcp_server.py
    config.py        # Settings loaded from environment variables
  ui/
    static/
      app.js         # Chat UI logic: SSE streaming, tab management, markdown rendering
      style.css      # Layout and theme
    templates/
      index.html     # Jinja2 base template (inlines static assets via FastAPI)
  mcp-server/
    er6_mcp_server.py   # Unchanged
  .env.example         # Documents required environment variables
```

No build toolchain. Plain HTML/CSS/JS served directly by FastAPI. `marked.js` (CDN) for markdown rendering.

## Backend

### FastAPI Routes

| Route | Method | Description |
|-------|--------|-------------|
| `/` | GET | Serve the UI (requires auth) |
| `/auth/login` | GET | Redirect to corporate IDP |
| `/auth/callback` | GET | Exchange code, set session cookie, redirect to `/` |
| `/auth/logout` | GET | Clear session cookie, redirect to IDP logout |
| `/chat` | POST | SSE stream — accepts conversation history, streams AI response chunks |
| `/health` | GET | Liveness check (no auth required) |

### `chat.py` — Streaming Loop

1. Receive conversation history as JSON in the POST body.
2. Build the Anthropic API request with:
   - Model: `claude-opus-4-7`
   - System prompt: contents of `CLAUDE.md`
   - Tool definitions: fetched from the running MCP server at startup
3. Stream the response. On each `tool_use` event, execute the tool via `mcp_client.py` and feed the result back as a `tool_result` message.
4. SSE-push text chunks to the browser as they arrive. Push a final `[DONE]` event on completion.
5. On error, push an `[ERROR]` event with a sanitized message.

### `mcp_client.py`

Manages a single long-lived subprocess running `er6_mcp_server.py` (stdio MCP protocol). Exposes `async call_tool(name, arguments) -> str`. Restarts the subprocess automatically if it exits unexpectedly.

### `auth.py`

Standard OIDC authorization code flow using `authlib`. Session state is a signed httponly cookie (key from `SESSION_SECRET` env var). On callback, store the user's `sub` and `preferred_username` claims in the session. All routes except `/health`, `/auth/login`, and `/auth/callback` require a valid session — redirect to `/auth/login` if missing.

### `config.py`

All secrets and deployment configuration via environment variables:

| Variable | Description |
|----------|-------------|
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `OIDC_CLIENT_ID` | Corporate IDP client ID |
| `OIDC_CLIENT_SECRET` | Corporate IDP client secret |
| `OIDC_DISCOVERY_URL` | IDP OpenID Connect discovery endpoint |
| `SESSION_SECRET` | Random secret for signing session cookies |
| `BASE_URL` | Public base URL of this app (e.g. `http://iam-assistant.internal`) |

## Frontend

### Layout

Two-panel layout, fixed height (`100vh`):

- **Header bar** (full width): App name, ER6 connection indicator (green dot = MCP subprocess alive), logged-in username, sign-out link.
- **Left panel (42%)**: Conversation history + input bar at the bottom.
- **Right panel (58%)**: Tabbed results workspace.

### Chat Panel

- Messages rendered in order, newest at the bottom, auto-scrolls.
- User messages right-aligned, AI messages left-aligned with an "AI" avatar.
- Streaming indicator: a blinking cursor appended to the last AI message while the SSE stream is open.
- Input: `<textarea>` that grows up to 4 lines; `Enter` sends, `Shift+Enter` inserts newline.
- Input is disabled while a stream is in progress.

### Results Panel

- Tab bar at the top. Each AI response that produces structured output (tables, findings, lists) opens a new tab. Tabs are named after the query subject (e.g. "BRT Coverage", "SoD Scan").
- Tab content is the AI response rendered as markdown via `marked.js`.
- Tables are rendered with sortable columns (client-side, no library needed — plain JS sort on `<th>` click).
- Finding callouts: blockquotes that start with `⚠` are styled as warning banners (amber left border).
- Tabs persist for the lifetime of the browser session. Maximum 10 tabs; oldest is dropped when exceeded.

### SSE Client (`app.js`)

1. On send, POST `/chat` with the full conversation history as JSON using `fetch()` with `Content-Type: application/json`.
2. Read the response as a `ReadableStream`, decoding SSE lines via a `TextDecoder`. (`EventSource` is not used — it only supports GET requests.)
3. On each `data:` line, append the chunk to the current AI message in the chat panel and to a buffer for the active results tab.
4. On `data: [DONE]`, finalize the markdown render in the results tab, re-enable the input.
5. On `data: [ERROR]`, show an inline error message in the chat panel, re-enable the input.

## Data Flow

```
Browser → POST /chat (conversation history JSON)
  → FastAPI chat.py
    → Anthropic API (streaming)
      ↔ tool_use events → mcp_client.py → er6_mcp_server.py → ER6
    → SSE chunks → Browser EventSource
      → Chat panel (streaming text)
      → Results tab (buffered, rendered on [DONE])
```

Conversation history is stored in the browser (`sessionStorage`) — not on the server. Each POST sends the full history. This keeps the server stateless beyond the session cookie and avoids needing a database for v1.

---

## Session & Security

- Session cookie: signed, httponly, `SameSite=Lax`. Contains `sub` and `preferred_username` from the OIDC token. No sensitive data stored in the cookie beyond identity.
- CSRF: mitigated by `SameSite=Lax` + checking `Origin` header on `/chat` POST.
- The `ANTHROPIC_API_KEY` and IDP credentials never leave the server.
- ER6 connection credentials (`.sapcli.env`) are read by `mcp_client.py` server-side, same as today.

## Deployment

Run with `uvicorn app.main:app --host 0.0.0.0 --port 8080` on any internal machine with the conda `sapcli-env` available. No Docker required for v1, but the app is stateless and containerisable.

Add to `.gitignore`: `.env`, `.superpowers/`

---

## Out of Scope (v1)

- Persistent conversation history across browser sessions
- Per-user conversation isolation on the server
- Export / download of results panels
- Mobile layout
- Role-based access control beyond "authenticated = can use"
